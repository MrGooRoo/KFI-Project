## 🔄 Процесс работы (7 этапов)

### [1/7] Извлечение текста из PDF
- PyMuPDF извлекает весь текст из документа
- Обрабатывает все страницы

### [2/7] Определение эмитента и периода
- Regex-паттерны ищут название организации
- Определяет период отчётности (квартал/год)
- Если не найдено → запрашивает вручную

### [3/7] Определение отраслевой группы
- Автоматический поиск по 7 группам ключевых слов
- Подсчёт совпадений для каждой группы
- Если не определено → выбор из списка

### [4/7] Извлечение финансовых данных
- 13 ключевых показателей из баланса, ОПУ, ОДДС
- Множественные regex-паттерны для каждого показателя
- Недостающие поля → ручной ввод

### [5/7] Расчёт КФИ
- Конвертация данных в формат калькулятора
- Расчёт 6 блоков (A-F) с учётом отраслевой группы
- Определение категории (Надёжный / Стабильный / и т.д.)

### [6/7] Генерация карточки
- HTML-карточка с детализацией
- PNG-изображение 480×920px
- Сохранение в `output/cards/`

### [7/7] Генерация текстов для соцсетей
- Telegram: полная версия с детализацией
- VK/Пульс: краткая версия
- Сохранение в `output/social_posts/`

## 🎯 Преимущества

### 1. Универсальность
- Работает с **любым эмитентом**
- Не требует предварительной настройки базы данных
- Подходит для разового анализа

### 2. Автоматизация
- Автоматическое извлечение названия и периода
- Автоопределение отраслевой группы
- Минимум ручного ввода

### 3. Скорость
- Полный анализ за 2-3 минуты
- От PDF до готового контента

### 4. Полнота
- Извлекает 13 финансовых показателей
- Генерирует карточку и тексты
- Сохраняет все данные в базу

### 5. Гибкость
- Можно дополнить данные вручную
- Выбор отрасли при необходимости
- Работает с любой структурой РСБУ

## 📊 Автоопределение отраслевой группы

### Ключевые слова

| Группа | Ключевые слова |
|--------|----------------|
| 1. Девелопмент | девелопмент, строительство, недвижимость, застройщик |
| 2. МФО | мфо, микрофинанс, микрокредит, займ, потребительский кредит |
| 3. Лизинг | лизинг, финансовая аренда |
| 4. Торговля | торговля, ритейл, магазин, сеть магазинов, розничная торговля |
| 5. Производство | производство, завод, фабрика, промышленность, изготовление |
| 6. Транспорт | транспорт, логистика, перевозка, доставка, грузоперевозки |
| 7. Прочие | холдинг, группа компаний, диверсифицированный |

### Алгоритм
1. Поиск ключевых слов в тексте PDF и названии эмитента
2. Подсчёт совпадений для каждой группы
3. Выбор группы с максимальным количеством совпадений
4. Если совпадений нет → ручной выбор из списка

## 🔧 Технические детали

### Класс StandalonePipeline

**Основные методы**:
- `run(pdf_path)` — главный метод запуска конвейера
- `_extract_text_from_pdf(pdf_path)` — извлечение текста через PyMuPDF
- `_extract_emitter_and_period(text)` — определение эмитента и периода
- `_generate_emitter_id(name)` — генерация ID через транслитерацию
- `_determine_industry_group(text, name)` — автоопределение отрасли
- `_extract_financial_data(text, id, period)` — извлечение финансовых данных
- `_fill_missing_fields(data)` — ручной ввод недостающих полей
- `_calculate_kfi(emitter, data, period)` — расчёт КФИ
- `_save_emitter_data(emitter, data, kfi)` — сохранение в базу
- `_generate_card(emitter_id)` — генерация карточки
- `_generate_social_text(emitter, kfi)` — генерация текстов

## ⚠️ Требования к PDF

### Обязательно
- ✅ Текстовый PDF (не скан)
- ✅ Возможность выделения текста
- ✅ Стандартная структура РСБУ

### Рекомендуется
- ✅ Официальный отчёт с e-disclosure.ru
- ✅ Полный комплект форм (баланс + ОПУ + ОДДС)
- ✅ Актуальный период отчётности

### Не поддерживается
- ❌ Сканы (изображения)
- ❌ Excel-файлы
- ❌ Нестандартные форматы отчётности

## 🚀 Готовность к веб-интерфейсу

Текущая реализация через терминал **полностью готова** к интеграции в веб:

### Будущий API endpoint

```python
@app.route('/api/analyze', methods=['POST'])
def analyze_emitter():
    # Получаем PDF из формы
    pdf_file = request.files['pdf']
    
    # Сохраняем временно
    temp_path = f"/tmp/{uuid.uuid4()}.pdf"
    pdf_file.save(temp_path)
    
    # Запускаем конвейер
    pipeline = StandalonePipeline()
    result = pipeline.run(temp_path)
    
    # Удаляем временный файл
    os.remove(temp_path)
    
    # Возвращаем результаты
    return jsonify({
        'success': True,
        'emitter': {
            'name': result['emitter']['name'],
            'sector': result['emitter']['sector'],
            'id': result['emitter']['emitter_id']
        },
        'kfi': {
            'score': result['kfi_result']['kfi_final'],
            'category': result['kfi_result']['category'],
            'blocks': result['kfi_result']['block_scores']
        },
        'outputs': {
            'card_html': f"/cards/{result['emitter']['emitter_id']}.html",
            'card_png': f"/cards/{result['emitter']['emitter_id']}.png",
            'telegram_text': result['social_text']['telegram'],
            'vk_text': result['social_text']['vk']
        }
    })
```
