# Автономный конвейер анализа эмитентов

**Версия**: 2.0  
**Дата**: 25 апреля 2026

---

## Концепция

Полностью автономный анализ эмитента по PDF-отчёту **без привязки к базе эмитентов**.

### Что делает конвейер

1. **Загружает PDF-отчёт** — любой эмитент, любая отчётность
2. **Извлекает данные**:
   - Название эмитента
   - Период отчётности
   - Все финансовые показатели (баланс, ОПУ, ОДДС)
3. **Определяет отраслевую группу** — автоматически по ключевым словам
4. **Рассчитывает КФИ** — по методологии v2.0
5. **Генерирует карточку** — HTML + PNG
6. **Создаёт тексты для соцсетей** — Telegram + VK/Пульс

---

## Использование

### Базовый запуск

```bash
python src/standalone_pipeline.py path/to/report.pdf
```

### Пример

```bash

```

## Результаты

### Итоговая сводка
```
======================================================================
  РЕЗУЛЬТАТЫ АНАЛИЗА
======================================================================

Эмитент: Кармани
Отрасль: МФО и потребкредитование
КФИ: 82.7 — Надёжный

Карточка: output/cards/card_karmani.html
          output/cards/card_karmani.png

Тексты для соцсетей:
  Telegram: output/social_posts/KARMANI_telegram.txt
  VK/Пульс: output/social_posts/KARMANI_vk.txt

======================================================================
```

### Что создаётся

1. **Карточка эмитента**:
   - `output/cards/card_<emitter_id>.html` — HTML-версия
   - `output/cards/card_<emitter_id>.png` — PNG-изображение (480×920px)

2. **Тексты для соцсетей**:
   - `output/social_posts/<EMITTER_ID>_telegram.txt` — полная версия
   - `output/social_posts/<EMITTER_ID>_vk.txt` — краткая версия

3. **Данные сохраняются**:
   - `data/financials/<emitter_id>_<period>.json` — финансовые данные
   - `data/calculations/<emitter_id>_<period>.json` — результаты расчёта КФИ
   - `data/emitters.json` — эмитент добавляется в базу

## Требования к PDF

### Формат
- **Текстовый PDF** (не скан)
- Возможность выделения текста
- Стандартная структура РСБУ отчётности

### Содержание
- Бухгалтерский баланс (Форма 1)
- Отчёт о финансовых результатах (Форма 2)
- Отчёт о движении денежных средств (Форма 4)

### Рекомендации
- Используйте официальные отчёты с e-disclosure.ru
- Проверьте, что PDF содержит текст (не изображение)
- Убедитесь, что отчёт полный (все 3 формы)

## Отличия от обычного конвейера

| Параметр | Обычный конвейер | Автономный конвейер |
|----------|------------------|---------------------|
| Эмитент | Должен быть в базе | Любой эмитент |
| Данные | Из базы или парсинг | Только из PDF |
| Отрасль | Из базы | Автоопределение |
| Использование | Регулярное обновление | Разовый анализ |

## Обработка ошибок

### PDF не читается
```
ERROR: Не удалось извлечь текст из PDF
```
**Решение**: Проверьте, что PDF текстовый (не скан)

### Не определился эмитент
```
Не удалось определить название эмитента. Введите вручную: Кармани
```

### Не определился период
```
Не удалось определить период. Введите вручную (например, 2024-Q4): 2024-Q4
```

### Мало данных извлечено
```
Извлечено полей: 3/13
```
**Решение**: Система запросит недостающие поля вручную

## Интеграция с веб-интерфейсом (будущее)

Текущая реализация через терминал легко адаптируется под веб:

```python
# Будущий API endpoint
@app.route('/api/analyze', methods=['POST'])
def analyze_emitter():
    pdf_file = request.files['pdf']
    
    # Сохраняем временно
    temp_path = save_temp_file(pdf_file)
    
    # Запускаем конвейер
    pipeline = StandalonePipeline()
    result = pipeline.run(temp_path)
    
    # Возвращаем результаты
    return jsonify({
        'kfi': result['kfi_result']['kfi_final'],
        'category': result['kfi_result']['category'],
        'card_url': f"/cards/{result['emitter']['emitter_id']}.png",
        'telegram_text': result['social_text']['telegram'],
        'vk_text': result['social_text']['vk']
    })
```
