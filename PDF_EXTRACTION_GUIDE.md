# Руководство по извлечению данных из PDF

## Установка зависимостей

```bash
pip install PyMuPDF
```

Или установите все зависимости:
```bash
pip install -r src/requirements.txt
```

## Использование в конвейере

### Автоматический режим (рекомендуется)

```bash
python src/pipeline.py CARMONEY
```

Если данных нет, конвейер предложит:
1. **Загрузить PDF-отчёт** ← выберите этот вариант
2. Ввести данные вручную
3. Пропустить

### Пример работы

```
[3/6] Проверка полноты данных...
      [WARN] Отсутствуют поля: revenue, equity, cfo

======================================================================
  ТРЕБУЕТСЯ ДОПОЛНЕНИЕ ФИНАНСОВЫХ ДАННЫХ
======================================================================

Варианты:
  1. Загрузить PDF-отчёт (автоматическое извлечение данных)
  2. Ввести данные вручную
  3. Пропустить (использовать демо-данные)

Выберите вариант (1/2/3): 1

Укажите путь к PDF-файлу: F:\Reports\carmoney_2024q4.pdf
```

## Что извлекается из PDF

### Баланс
- Оборотные активы
- Денежные средства
- Запасы
- Всего активов
- Краткосрочные обязательства
- Краткосрочный долг
- Долгосрочный долг
- Капитал и резервы

### Отчёт о прибылях и убытках
- Выручка
- Операционная прибыль (EBIT)
- Чистая прибыль
- Процентные расходы

### Отчёт о движении денежных средств
- Денежный поток от операционной деятельности (CFO)
- Капитальные затраты (CAPEX)

## Поддерживаемые форматы отчётности

- РСБУ (Российские стандарты бухгалтерского учёта)
- Форма 1 (Бухгалтерский баланс)
- Форма 2 (Отчёт о финансовых результатах)
- Форма 4 (Отчёт о движении денежных средств)

## Ручной ввод недостающих полей

Если PDF-экстрактор не нашёл все поля, система автоматически запросит только недостающие:

```
Не найденные поля:
  ✗ Капитальные затраты
  ✗ Выручка предыдущего периода

======================================================================
  РУЧНОЙ ВВОД НЕДОСТАЮЩИХ ПОЛЕЙ
======================================================================

Капитальные затраты (тыс. руб.): 50000
Выручка предыдущего периода (тыс. руб.): 1100000

[OK] Данные дополнены и сохранены
```

## Автономное использование PDF-экстрактора

Можно использовать экстрактор отдельно от конвейера:

```bash
python src/pdf_financial_extractor.py reports/carmoney_2024q4.pdf CARMONEY 2024-Q4
```

Результат:
- Извлечённые данные выводятся в консоль
- Предлагается сохранить в `data/financials/`

## Советы по подготовке PDF

1. **Используйте текстовые PDF** (не сканы)
   - Экстрактор работает с текстом, а не с изображениями
   - Если у вас скан, используйте OCR перед загрузкой

2. **Проверьте структуру отчёта**
   - Должны быть стандартные названия строк (Выручка, Баланс, и т.д.)
   - Числа должны быть в формате с пробелами: `1 234 567`

3. **Укажите единицы измерения**
   - Экстрактор ожидает значения в тысячах рублей
   - Если в отчёте рубли, система автоматически конвертирует

## Известные ограничения

1. **Не работает со сканами** — требуется текстовый PDF
2. **Может не распознать нестандартные названия строк** — в этом случае используйте ручной ввод
3. **Не извлекает данные из таблиц Excel** — только PDF

## Что делать, если экстрактор не работает

1. Проверьте, что PDF содержит текст (попробуйте выделить текст в Adobe Reader)
2. Убедитесь, что отчёт в формате РСБУ
3. Используйте вариант 2 (ручной ввод) в конвейере
4. Или используйте `financial_data_manager.py` напрямую:
   ```bash
   python src/financial_data_manager.py CARMONEY
   ```
