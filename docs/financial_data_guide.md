# Руководство по работе с финансовыми данными

**Дата**: 25 апреля 2026  
**Версия**: 1.0

---

## Обзор

Система сбора финансовых данных для КФИ-проекта включает три компонента:

1. **edisclosure_scraper.py** — автоматический парсинг e-disclosure.ru (Playwright)
2. **financial_data_manager.py** — улучшенный ручной ввод с валидацией
3. **financial_parser.py** — интеграция всех источников

---

## Структура данных

Финансовые данные хранятся в `data/financials/` в формате:
```
{emitter_id}_{period}.json
```

Примеры:
- `CARMONEY_2024-Q4.json`
- `GTLK_2024-12.json`

---

## Использование

### 1. Ручной ввод данных

```bash
python src/financial_data_manager.py CARMONEY
```

Система запросит:
- Период отчётности (2024-Q4, 2024-12)
- Баланс (форма 1): 9 показателей
- ОПУ (форма 2): 4 показателя
- ОДДС (форма 4): 2 показателя
- Предыдущие периоды: 2 показателя
- Отраслевые показатели (зависит от группы)
- Метаданные (источник, автор, примечания)

**Важно**: Все значения вводятся в **тысячах рублей**.

### 2. Просмотр сохранённых данных

```bash
# Все данные
python src/financial_data_manager.py list

# Данные конкретного эмитента
python src/financial_data_manager.py list CARMONEY
```

### 3. Автоматический парсинг (экспериментально)

```bash
python src/financial_parser.py CARMONEY
```

Система попробует:
1. Playwright скрапер (e-disclosure.ru)
2. API e-disclosure (если доступен)
3. Ручной ввод (fallback)

### 1. e-disclosure.ru

**URL**: https://www.e-disclosure.ru/portal/company.aspx?id={company_id}

**Разделы**:
- Годовая бухгалтерская отчётность
- Квартальная отчётность

## Расчёт коэффициентов

После ввода данных система автоматически рассчитывает:

### Блок A: Платёжная устойчивость
- Current Ratio = Оборотные активы / Краткосрочные обязательства
- Quick Ratio = (Оборотные активы - Запасы) / Краткосрочные обязательства
- Cash Ratio = (Денежные средства + Краткосрочные вложения) / Краткосрочные обязательства
- ICR = Прибыль от продаж / Проценты к уплате

### Блок B: Структурная устойчивость
- Debt/EBITDA = (Долгосрочный долг + Краткосрочный долг) / EBITDA
- D/E Ratio = Долг / Собственный капитал
- Equity Ratio = Собственный капитал / Активы

### Блок C: Рентабельность
- ROE = Чистая прибыль / Собственный капитал
- ROA = Чистая прибыль / Активы
- Operating Margin = Прибыль от продаж / Выручка

### Блок D: Денежный поток
- FCF/EBITDA = (CFO - CAPEX) / EBITDA
- OCF Growth = (CFO - CFO_prev) / CFO_prev
- CAPEX Coverage = CFO / CAPEX

---

## Валидация данных

Система проверяет:

1. **Логические связи**:
   - Активы = Обязательства + Капитал
   - Оборотные активы ≥ Денежные средства
   - Краткосрочные обязательства ≥ Краткосрочные займы

2. **Разумные диапазоны**:
   - ROE: -50% до +100%
   - Debt/Equity: 0 до 20
   - Current Ratio: 0.1 до 10

3. **Обязательные поля**:
   - Выручка, Активы, Капитал, Обязательства

## Интеграция с расчётом КФИ

После ввода данных запустите расчёт:

```bash
python src/main.py calculate CARMONEY
```

Система:
1. Загрузит финансовые данные из `data/financials/`
2. Рассчитает коэффициенты
3. Вычислит баллы по блокам A-D
4. Сохранит результат в `data/calculations/`

## Частые проблемы

### 1. "Эмитент не найден"

**Причина**: Эмитент отсутствует в `data/emitters.json`

**Решение**: Добавьте эмитента в базу

### 2. "Данные уже существуют"

**Причина**: Файл для этого периода уже создан

**Решение**: Подтвердите перезапись или выберите другой период

### 3. "Некорректное число"

**Причина**: Неправильный формат ввода

**Решение**: Используйте только цифры, точку или запятую для дробной части
