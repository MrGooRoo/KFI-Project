# Конвейер обработки эмитента

**Файл**: `src/pipeline.py`  
**Дата создания**: 2026-04-25

---

## 📋 Описание

Автоматизированный конвейер для полной обработки эмитента от сбора данных до публикации в социальных сетях.

---

## 🔄 Этапы конвейера

### 1. Выбор эмитента
- Интерактивный выбор из списка
- Или указание ID через CLI

### 2. Парсинг данных
- Попытка получить данные с MOEX API
- Проверка существующих файлов
- (Опционально) Парсинг e-disclosure.ru

### 3. Проверка полноты данных
- Автоматическая проверка критических полей
- Если данных недостаточно → запуск ручного ввода
- Использование `FinancialDataManager` для интерактивного ввода

### 4. Расчёт КФИ
- Конвертация данных в формат калькулятора
- Расчёт всех 6 блоков (A-F)
- Применение корректировок и флагов

### 5. Генерация карточки
- HTML карточка с детализацией
- PNG изображение для публикации
- Сохранение в `output/cards/`

### 6. Генерация текста для соцсетей
- **Telegram**: полный разбор с детализацией
- **VK/Пульс**: краткая версия
- Сохранение в `output/social_posts/`

---

## 🚀 Использование

### Режим 1: CLI с указанием эмитента
```bash
# Обработать эмитента (текущий период)
python src/pipeline.py CARMONEY

# Обработать эмитента за конкретный период
python src/pipeline.py GTLK 2024-Q4

# Пропустить парсинг, использовать существующие данные
python src/pipeline.py SETL --skip-parsing
```

### Режим 2: Интерактивный
```bash
python src/pipeline.py --interactive
```

Интерактивный режим:
1. Показывает список всех эмитентов
2. Позволяет выбрать по номеру или ID
3. Запрашивает период (или использует текущий)
4. Запускает полный конвейер

## 🔧 Технические детали

### Класс EmitterPipeline

**Методы**:
- `run(emitter_id, period, skip_parsing)` — запуск полного конвейера
- `_get_emitter(emitter_id)` — получение данных эмитента
- `_parse_data(emitter, period)` — парсинг финансовых данных
- `_ensure_complete_data(...)` — проверка и ручной ввод
- `_calculate_kfi(...)` — расчёт КФИ
- `_generate_card(emitter_id)` — генерация карточки
- `_generate_social_text(...)` — генерация текстов

### Зависимости
```python
from moex_parser import MOEXBondParser, load_emitters
```

## 🎯 Примеры использования

```bash
python src/pipeline.py NEWEMITTER
```
1. Парсинг не найдёт данных
2. Запустится ручной ввод
3. После ввода — расчёт и генерация

### Пример 2: Обновление существующего
```bash
python src/pipeline.py CARMONEY 2024-Q4
```
1. Найдёт существующие данные
2. Пересчитает КФИ
3. Обновит карточку и тексты

### Пример 3: Пакетная обработка
```bash
# Создать скрипт для обработки всех эмитентов
for emitter in CARMONEY GTLK SETL; do
    python src/pipeline.py $emitter
done
```

## 🔄 Интеграция с другими модулями


### С веб-интерфейсом (будущее)
```python
# API endpoint для запуска конвейера
@app.route('/api/process/<emitter_id>')
def process_emitter(emitter_id):
    pipeline = EmitterPipeline()
    result = pipeline.run(emitter_id)
    return jsonify(result)
```

## ✅ Преимущества конвейера

1. **Автоматизация**: Один скрипт вместо 5 отдельных команд
2. **Интерактивность**: Запрос данных только когда нужно
3. **Гибкость**: Можно пропустить этапы или использовать существующие данные
4. **Полнота**: От данных до готового контента для публикации
5. **Удобство**: Интерактивный режим для новичков, CLI для автоматизации
