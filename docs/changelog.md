## [2.0.0] — Январь 2025

### Добавлено
- Полная документация методологии KFI v2.0 (methodology_v2.md)
- 7 отраслевых групп с адаптированными нормами
- Блок A — Платёжная устойчивость (вес 20%) с отраслевыми вариантами
- Блок B — Структурная устойчивость (вес 25%) с K_escrow для девелопмента
- Блок C — Рентабельность (вес 10%)
- Блок D — Денежный поток (вес 25%) с четырьмя показателями D1–D4
- Блок E — Облигационный профиль (вес 15%), уникальный блок для ВДО
- Блок F — Качественная оценка (вес 5%), гибридная модель нейросеть+редактор
- Финальная формула КФИ с системой корректировок
- Три правила корректировки: жёсткий кэп, штраф двух слабых блоков, флаги
- Система флагов риска: bankruptcy_risk, revenue_quality_risk,
  bond_concentration_risk, data_missing, young_issuer, ebitda_approximated
- Шкала пяти категорий: Надёжный / Стабильный / Умеренный / Высокий / Критический
- Схема базы данных (7 ключевых сущностей)
- Контентная модель: 4 типа постов для TG и Пульса
- Поэтапный план монетизации
- Технический стек: Python + Flask, SQLite->PostgreSQL, Claude API, n8n, Jino VPS

### Изменено
- Методология переработана с v1.0: добавлена отраслевая сегментация
- Вес блока F снижен с 10% до 5% (субъективность ограничена)
- Блок E выделен как уникальный элемент продукта
- Добавлен K_escrow вместо простого вычитания эскроу из долга

### Исправлено
- Устранена проблема некорректного применения Альтмана и Таффлера
- Устранено отсутствие отраслевой сегментации в v1.0
- Добавлена политика обработки отсутствующих данных

## [1.0.0] — 2024

### Добавлено
- Первоначальная концепция КФИ
- Базовая архитектура из 6 блоков
- Принцип слабого звена
- README и структура репозитория
- Заготовки файлов в src/

### Известные проблемы (устранены в v2.0)
- Отсутствует отраслевая сегментация
- Некорректное применение Z-моделей (Альтман, Таффлер)
- Не описан механизм перевода показателей в баллы 0–100
- Нет политики обработки отсутствующих данных
- Блок качественной оценки не детализирован
