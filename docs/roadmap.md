## ФАЗА 1 — MVP-0: Данные [ПЛАНИРУЕТСЯ]

Цель: получить реальные данные для тестирования системы.  
Срок: 1–2 недели.

### Задачи
- [ ] Выбрать 5–10 эмитентов для первого расчёта
- [ ] Собрать РСБУ-отчётность вручную с e-disclosure.ru
- [ ] Рассчитать блоки A, B, C, D вручную по формулам
- [ ] Получить данные блока E через MOEX API вручную
- [ ] Заполнить блок F вручную с верификацией
- [ ] Сохранить данные в JSON-файлы по структуре БД
- [ ] Проверить логику правил корректировки на реальных данных
- [ ] Откалибровать пороги если нужно

### Критерий готовности
Есть 5–10 корректно рассчитанных КФИ в JSON-формате, 
готовых к загрузке в БД.

## Принципы приоритизации

1. Методология первична — код пишется только после фиксации методологии
2. Данные первичны — сайт строится только когда есть реальные данные
3. Простота MVP — лучше работающее простое, чем сложное незавершённое
4. Автоматизация постепенно — сначала полуавтомат, потом полный автомат
5. Качество важнее скорости — репутация строится на точности данных
