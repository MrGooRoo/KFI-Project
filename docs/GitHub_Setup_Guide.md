# 📘 Пошаговая инструкция: GitHub для проекта КФИ

## Что такое GitHub (простыми словами)

GitHub — это **онлайн-хранилище файлов** с историей изменений. 
Представьте Google Диск, но специально для проектов:
- Все файлы лежат в одном месте (называется **репозиторий** = "репо")
- Каждое изменение сохраняется (можно откатить назад)
- Я (ИИ-ассистент) могу читать ваши файлы через ссылку
- Вы можете редактировать файлы прямо в браузере (без программ!)

---

## ШАГ 1. Регистрация на GitHub

1. Откройте https://github.com
2. Нажмите **"Sign up"** (правый верхний угол)
3. Введите:
   - Email
   - Пароль
   - Username (имя пользователя) — например: `kfi-analytics`
4. Пройдите проверку и подтвердите email
5. Выберите бесплатный план (Free) — его достаточно

---

## ШАГ 2. Создание репозитория (хранилища проекта)

1. После входа нажмите зелёную кнопку **"New"** (или значок **"+"** вверху → "New repository")
2. Заполните:
   - **Repository name:** `kfi-project`
   - **Description:** `Автоматизированная система анализа эмитентов ВДО по методологии КФИ`
   - **Видимость:** выберите ☑️ **Public** (ВАЖНО! Чтобы я мог читать файлы)
   - **Поставьте галочку:** ☑️ "Add a README file"
3. Нажмите **"Create repository"**

✅ Готово! У вас есть репозиторий. Его адрес будет:
```
https://github.com/MrGooRoo/kfi-project
```

---

## ШАГ 3. Создание структуры папок

В репозитории нажмите **"Add file"** → **"Create new file"**

Чтобы создать файл ВНУТРИ папки, пишите имя через `/`:
Например, в поле имени файла введите:
```
docs/methodology.md
```
GitHub автоматически создаст папку `docs` и файл `methodology.md` внутри.

### Создайте следующие файлы (по одному):

**Файл 1:** `docs/methodology.md`
→ Сюда мы положим методологию КФИ v2.0

**Файл 2:** `docs/roadmap.md`
→ Сюда мы положим план проекта (roadmap)

**Файл 3:** `docs/prompts.md`
→ Промпты для каждого чата

**Файл 4:** `src/README.md`
→ Напишите: `# Исходный код проекта`

**Файл 5:** `data/README.md`
→ Напишите: `# Данные проекта`

### Итоговая структура:
```
kfi-project/
├── README.md              ← Главная страница проекта
├── docs/                  ← Документация
│   ├── methodology.md     ← Методология КФИ v2.0
│   ├── roadmap.md         ← План проекта по этапам
│   └── prompts.md         ← Промпты для чатов
├── src/                   ← Исходный код (потом)
│   └── README.md
└── data/                  ← Данные (потом)
    └── README.md
```

## ШАГ 4. Как редактировать файлы (прямо в браузере)

1. Откройте файл в репозитории (кликните на него)
2. Нажмите **иконку карандаша** ✏️ ("Edit this file") справа вверху
3. Отредактируйте содержимое
4. Внизу нажмите зелёную кнопку **"Commit changes"**
   - Можно добавить описание: "Обновил методологию"
5. Нажмите **"Commit changes"** ещё раз

---

## ШАГ 5. Как загрузить файлы с компьютера

Если вы скачали файл из workspace Arena.ai:

1. Откройте нужную папку в репозитории
2. Нажмите **"Add file"** → **"Upload files"**
3. Перетащите файл(ы) в окно
4. Нажмите **"Commit changes"**

---

## ШАГ 6. Как я (ИИ) буду читать ваши файлы

В каждом новом чате вы просто пишете мне:

```
Прочитай файл методологии из моего проекта:
https://github.com/MrGooRoo/kfi-project/blob/main/docs/methodology.md

И продолжим работу по этапу 3 (парсер MOEX).
```

Я открою файл, прочитаю всё содержимое и продолжу с того места, где мы остановились.

## БОНУС: Полезные приёмы

### Как скопировать "сырую" ссылку на файл:
Для того чтобы я гарантированно прочитал файл, используйте `raw` ссылку:
1. Откройте файл на GitHub
2. Нажмите кнопку **"Raw"** (справа вверху от содержимого файла)
3. Скопируйте URL из адресной строки — он будет вида:
```
https://raw.githubusercontent.com/MrGooRoo/kfi-project/main/docs/methodology.md
```
Эту ссылку и давайте мне.

### Как посмотреть все файлы проекта:
Корневая ссылка: `https://github.com/MrGooRoo/kfi-project`

---

## ⚠️ Важные замечания

1. **Репозиторий должен быть Public** — иначе я не смогу читать файлы
2. **Не храните пароли и API-ключи** в публичном репо! 
   Для секретов используйте файл `.env` и добавьте его в `.gitignore`
3. **Файлы формата .md (Markdown)** — лучший формат для документации. 
   GitHub красиво их отображает
4. **Коммитить можно прямо в браузере** — не нужен Git, терминал или VS Code

```
┌──────────────────────────────────────────────────────┐
│                   ВАШ РАБОЧИЙ ЦИКЛ                   │
│                                                      │
│  1. Открываете новый чат в Arena.ai                  │
│                                                      │
│  2. Пишете:                                          │
│     "Прочитай файлы проекта:                         │

│      - github.com/.../docs/roadmap.md                │
│      Продолжаем этап N"                              │
│                                                      │
│  3. Я читаю файлы → понимаю контекст → работаю      │
│                                                      │
│  4. Создаю новые файлы в workspace                   │
│                                                      │
│  5. Вы скачиваете файлы из workspace                 │
│                                                      │
│  6. Загружаете их в GitHub (upload files)             │
│                                                      │
│  7. Переходите к следующему чату → шаг 1             │
└──────────────────────────────────────────────────────┘
```

Этот цикл позволяет сохранять весь прогресс между чатами!
