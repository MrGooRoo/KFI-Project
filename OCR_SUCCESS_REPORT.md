## 🔧 Технические улучшения

### 1. Улучшенный парсинг OCR-текста

**Файл**: `src/enhanced_pdf_extractor.py`

**Изменения**:
```python
# Было: простое разбиение по пробелам
cells = re.split(r'\s{2,}|\t', line)

# Стало: умный парсинг с разделением текста и чисел
for part in parts:
    part_clean = part.replace(' ', '').replace('\xa0', '')
    if re.match(r'^-?\d+[\d\s]*$', part_clean):
        numbers.append(part_clean)
    else:
        text_parts.append(part)
```

**Результат**: Количество извлечённых строк таблиц увеличилось с 1-3 до 15-45 на страницу.

### 2. Нечёткое сопоставление (Fuzzy Matching)

**Добавлен метод**: `_fuzzy_match(pattern, text)`

```python
def _fuzzy_match(self, pattern: str, text: str) -> bool:
    pattern_words = pattern.split()
    matches = sum(1 for word in pattern_words if len(word) >= 4 and word in text)
    return matches > 0 and matches >= len(pattern_words) / 2
```

**Результат**: Парсер теперь находит показатели даже при OCR-ошибках (замена букв, лишние пробелы).

### 3. Расширенные маппинги

**Добавлено вариантов названий**:
- Баланс: 8 → 15 вариантов
- ОПУ: 6 → 12 вариантов
- ОДДС: 3 → 7 вариантов

**Примеры**:
```python
'итого по разделу ii': 'current_assets',
'итого раздел ii': 'current_assets',
'оборотные активы итого': 'current_assets',
```

### 4. Исправление обработки None

**Было**:
```python
'debt_ebitda': (financial_data.long_term_debt + financial_data.short_term_debt) / ...
# TypeError: unsupported operand type(s) for +: 'NoneType' and 'float'
```

**Стало**:
```python
'debt_ebitda': ((financial_data.long_term_debt or 0) + (financial_data.short_term_debt or 0)) / ...
```

## ⚠️ Известные ограничения

### 1. Не все поля извлекаются

**Не найдено**:
- Долгосрочный долг (long_term_debt)
- Денежный поток от операций (cfo)

**Причины**:
- Показатели могут быть на других страницах
- OCR может не распознать специфические форматы
- Названия строк могут отличаться от маппингов

**Решение**: Добавить больше вариантов названий в маппинги.

### 2. Некоторые значения могут быть неточными

**Пример**: `total_assets = 2,025` (слишком мало для крупной компании)

**Причина**: OCR может неправильно распознать разряды числа (2,025 млн → 2,025 тыс.)

**Решение**: Добавить валидацию и автоматическую корректировку масштаба.

### 3. Время обработки

**Текущее**: ~3 минуты на 76 страниц (2.4 сек/страница)

**Оптимизация**:
- Обрабатывать только страницы с таблицами (определять по содержимому)
- Использовать параллельную обработку страниц
- Кэшировать результаты OCR
