## Что изменилось

### Новая функциональность

Конвейер теперь поддерживает **автоматическое извлечение данных из PDF-отчётов**.

Вместо полностью ручного ввода данных, система предлагает 3 варианта:

1. **Загрузить PDF-отчёт** — автоматическое извлечение данных
2. **Ручной ввод** — интерактивный ввод всех полей
3. **Пропустить** — использовать демо-данные

## Технические детали

### PDF-экстрактор

**Класс**: `PDFFinancialExtractor`

**Методы**:
- `extract_from_pdf(pdf_path, emitter_id, period)` — основной метод извлечения
- `_parse_financial_data(text, emitter_id, period)` — парсинг текста отчёта
- `_extract_value(text, patterns)` — извлечение числовых значений по regex
- `_log_extracted_fields(data)` — вывод результатов извлечения

**Что извлекается**:

| Раздел | Показатели |
|--------|-----------|
| Баланс | Оборотные активы, денежные средства, запасы, всего активов, краткосрочные обязательства, долг, капитал |
| ОПУ | Выручка, операционная прибыль, чистая прибыль, процентные расходы |
| ОДДС | Денежный поток от операций (CFO), капитальные затраты (CAPEX) |

**Технология**:
- PyMuPDF (fitz) для извлечения текста из PDF
- Регулярные выражения для поиска финансовых показателей
- Множественные паттерны для каждого показателя (учитывает разные формулировки)

### Интеграция в конвейер

**Изменения в `pipeline.py`**:

1. Добавлен импорт `PDFFinancialExtractor`
2. Инициализация экстрактора в `__init__()`
3. Переработан метод `_ensure_complete_data()`:
   - Добавлено меню выбора (PDF / ручной ввод / пропустить)
   - Вызов PDF-экстрактора при выборе варианта 1
   - Проверка полноты извлечённых данных
   - Запрос недостающих полей вручную
4. Добавлены вспомогательные методы:
   - `_manual_input_missing_fields()` — ввод только недостающих полей
   - `_fallback_to_manual_input()` — запасной вариант при ошибке PDF

## Поддерживаемые форматы

- **РСБУ** (Российские стандарты бухгалтерского учёта)
- **Форма 1** — Бухгалтерский баланс
- **Форма 2** — Отчёт о финансовых результатах
- **Форма 4** — Отчёт о движении денежных средств

**Требования к PDF**:
- Текстовый формат (не скан)
- Стандартные названия строк отчётности
- Числа в формате с пробелами: `1 234 567`

---

## Известные ограничения

1. **Не работает со сканами** — требуется текстовый PDF с возможностью выделения текста
2. **Может не распознать нестандартные названия** — используйте ручной ввод для таких случаев
3. **Не поддерживает Excel** — только PDF-файлы
4. **Зависит от структуры отчёта** — лучше всего работает со стандартными формами РСБУ

---

## Преимущества

1. **Экономия времени** — автоматическое извлечение 10-13 показателей вместо ручного ввода
2. **Снижение ошибок** — меньше ручного ввода = меньше опечаток
3. **Гибкость** — можно дополнить недостающие поля вручную
4. **Прозрачность** — система показывает, что извлечено, а что нет

## Тестирование

Для тестирования потребуется:
1. PDF-отчёт любого эмитента (РСБУ, текстовый формат)
2. Запуск конвейера: `python src/pipeline.py EMITTER_ID`
3. Выбор варианта 1 (загрузить PDF)
4. Проверка извлечённых данных

**Рекомендуемые эмитенты для тестирования**:
- CARMONEY (МФО)
- GTLK (Лизинг)
- SETL (Торговля)
