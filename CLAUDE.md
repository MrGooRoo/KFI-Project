# КФИ Project — Документация для разработки

> Кредитно-Финансовый Индекс — автоматизированная система анализа эмитентов высокодоходных облигаций

## О проекте

**Цель**: Создать публичную систему оценки надёжности эмитентов ВДО для частных инвесторов.

**Методология**: КФИ v2.0 — интегральный индекс 0-100 на основе 6 блоков анализа с отраслевой сегментацией (7 групп).

**Текущий статус**: MVP-0 завершён. Работает расчётный движок, генератор карточек, базовая интеграция с MOEX API.

## Методология КФИ v2.0

### Структура индекса

КФИ = 0.20×A + 0.25×B + 0.10×C + 0.25×D + 0.15×E + 0.05×F

| Блок | Название | Вес | Описание |
|------|----------|-----|----------|
| A | Платёжная устойчивость | 20% | Ликвидность, ICR |
| B | Структурная устойчивость | 25% | Долговая нагрузка, капитал |
| C | Рентабельность | 10% | ROE, маржинальность |
| D | Денежный поток | 25% | FCF, Cash Conversion |
| E | Облигационный профиль | 15% | Параметры выпуска, история |
| F | Качественная оценка | 5% | Нефинансовые факторы |

### Отраслевые группы

1. Девелопмент и строительство
2. МФО и потребкредитование
3. Лизинг
4. Торговля и ритейл
5. Производство и промышленность
6. Транспорт и логистика
7. Прочие и диверсифицированные

Каждая группа имеет адаптированные нормы для блоков A и B.

### Шкала категорий

| КФИ | Категория | Цвет |
|-----|-----------|------|
| 80-100 | Надёжный | Зелёный |
| 60-79 | Стабильный | Синий |
| 40-59 | Умеренный риск | Жёлтый |
| 20-39 | Высокий риск | Оранжевый |
| 0-19 | Критический | Красный |

## Архитектура кода

### main.py — Главный пайплайн

Управляет всеми этапами:
- `collect` — сбор данных (РСБУ + MOEX)
- `calculate` — расчёт КФИ
- `generate` — генерация карточек
- `full` — полный цикл

### moex_parser.py — MOEX ISS API

**Класс**: `MOEXBondParser`

**Методы**:
- `get_market_data(isin, board_id)` — рыночные данные по ISIN
- `fetch_emitter(emitter)` — данные по всем облигациям эмитента
- `load_emitters()` — загрузка базы эмитентов

**Источник**: https://iss.moex.com/iss/engines/stock/markets/bonds/

### financial_parser.py — Финансовая отчётность

**Функции**:
- `load_financial_data(emitter_id)` — загрузка сохранённых данных
- `rsbu_to_financials(raw_data)` — конвертация РСБУ в показатели

**Статус**: Частично реализован. Требуется доработка парсинга e-disclosure.ru.

### kfi_calculator.py — Расчётный движок

**Класс**: `KfiCalculator`

**Методы**:
- `score_block_a/b/c/d/e/f(...)` — расчёт баллов по блокам
- `calculate_kfi(emitter, financials, period)` — полный расчёт КФИ
- `save_calculations(path)` — сохранение результатов

**Логика**:
1. Расчёт баллов по каждому блоку (0-100)
2. Взвешенная сумма → КФИ_base
3. Применение корректировок и флагов риска
4. Определение категории

### card_generator.py — Генератор карточек

**Класс**: `CardGenerator`

**Методы**:
- `generate_card(emitter_id)` — генерация HTML + PNG для эмитента
- `generate_all()` — генерация для всех эмитентов
- `load_financials(emitter_id)` — загрузка финансовых данных
- `enrich_block_e(emitter)` — обогащение данными MOEX

**Шаблон**: `card_template.html` (Jinja2)

**Вывод**: `output/cards/card_{emitter_id}.html` и `.png`

## Известные проблемы

### 1. Автоматический парсинг финансовых данных

**Проблема**: e-disclosure.ru блокирует автоматические запросы (403 Forbidden).

**Текущее решение**: 
- ✅ Создан `financial_data_manager.py` — улучшенный ручной ввод с валидацией
- ✅ Создан `edisclosure_scraper.py` — Playwright скрапер (экспериментально)
- ✅ Структурированное хранение в `data/financials/`

**Статус**: Частично решено. Ручной ввод работает, автоматический парсинг требует доработки.

**Приоритет**: Высокий

**Документация**: `docs/financial_data_guide.md`

### 2. MOEX API возвращает пустые данные

**Проблема**: Для некоторых ISIN не удаётся получить YTM и цены.

**Причина**: Возможно, неправильный board_id или облигация не торгуется.

**Решение**: Добавить fallback на другие boards (TQOB, TQIR).

**Приоритет**: Средний

### 3. Кодировка в консоли Windows

**Проблема**: Эмодзи не отображаются корректно в cmd.

**Решение**: Заменены на ASCII-маркеры ([OK], [ERROR], [WARN]).

**Статус**: Исправлено
